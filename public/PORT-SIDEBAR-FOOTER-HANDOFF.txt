PORT SIDEBAR + FOOTER CHANGES TO A SIMILAR CODEBASE
==================================================

Purpose
-------
This file explains exactly what was changed so another Cursor instance can repeat the same setup in a similar project.

Main goal:
1. The sidebar should live in its own repo/submodule.
2. Bringing in that sidebar should also bring in the footer behavior.
3. The app should only pass a footer background URL and point to the submodule.


BIG PICTURE
-----------
Before:
- The old local sidebar had a footer band under the main artboard.
- The new external `waypoint-sidebar` version did not include that footer.

After:
- The footer was added into the sidebar repo itself.
- The app now uses the submodule version of the sidebar.
- The app passes `footerBackgroundUrl="/news_bg.jpg"` into `LunaSidebar`.

This means:
- If the sidebar repo changes, the footer can change too.
- Updating the submodule updates both sidebar and footer behavior together.

In the app repo:
- `src/App.tsx`

In the sidebar repo / submodule:

STEP 1: MAKE THE APP IMPORT THE SIDEBAR FROM THE SUBMODULE
----------------------------------------------------------
If the app used to import a local sidebar like this:

```ts
import { LunaSidebar } from '../luna-sidebar/index.js'
```

change it to the submodule source path:

```ts
import { LunaSidebar } from '../luna-sidebar/src/luna-sidebar/index.js'
```


STEP 2: PASS THE FOOTER BACKGROUND URL FROM THE APP
---------------------------------------------------
Update the app's `LunaSidebar` usage so it passes a footer image URL.

Final `src/App.tsx`:

```tsx
import { LunaSidebar } from '../luna-sidebar/src/luna-sidebar/index.js'
import { FLOW_SIDEBAR_ITEMS } from './flowSidebarItems'
import { useFlowStep, useFlowStore } from './store/flowStore'
import './App.css'

function App() {
  const { step } = useFlowStep()
  const goToStepById = useFlowStore((s) => s.goToStepById)

  return (
    <LunaSidebar
      items={FLOW_SIDEBAR_ITEMS}
      footerBackgroundUrl="/news_bg.jpg"
      initialActiveId={step.id}
      onActiveItemChange={(id) => {
        const hit = FLOW_SIDEBAR_ITEMS.find((item) => item.id === id)
        if (hit) goToStepById(hit.id)
      }}
      railLabel="FLOW"
    />
  )
}

export default App
```

Important:
- `footerBackgroundUrl` points at a file inside the app's `public/` folder.
- In this project that file is `public/news_bg.jpg`.


STEP 3: ADD FOOTER SIZE HELPERS TO canvasScale.js
-------------------------------------------------
In the sidebar repo's `src/luna-sidebar/canvasScale.js`, add a footer rem token and a helper that returns footer height in pixels.

Final `canvasScale.js`:

```js
/**
 * Canvas scaling — keep DESIGN_REM_H in sync with `--canvas-h` in LunaSidebar.css :root.
 * Design token: 1rem = 16 at default root → CANVAS_W/H = DESIGN_REM_* × DESIGN_ROOT_PX.
 */

export const DESIGN_REM_W = 160;
/** Must match `--canvas-h` in LunaSidebar.css (rem). 1440÷16 = 90rem. */
export const DESIGN_REM_H = 90;
export const DESIGN_ROOT_PX = 16;

export const CANVAS_W = DESIGN_REM_W * DESIGN_ROOT_PX;
export const CANVAS_H = DESIGN_REM_H * DESIGN_ROOT_PX;

/** Must match `--page-row-otf-footer-h` in LunaSidebar.css (`:root`). */
export const OTF_FOOTER_DESIGN_REM = 48;

/** Design-pixel height of the OTF footer band (below main canvas). */
export function getOtfFooterDesignHeightPx() {
  return OTF_FOOTER_DESIGN_REM * DESIGN_ROOT_PX;
}

export function getViewportSize() {
  if (typeof window === "undefined") {
    return { width: CANVAS_W, height: CANVAS_H };
  }
  const vv = window.visualViewport;
  if (vv) {
    return { width: vv.width, height: vv.height };
  }
  return { width: window.innerWidth, height: window.innerHeight };
}

/** “Contain” scale: fit full CANVAS_W×CANVAS_H design in the viewport. */
export function getCanvasContainScale(width, height) {
  const sx = width / CANVAS_W;
  const sy = height / CANVAS_H;
  return Math.min(sx, sy);
}
```

STEP 8: MAKE SURE THE APP CAN SCROLL TO THE FOOTER
--------------------------------------------------
If the app root forces a single full-height child with no vertical overflow, the footer can exist but stay unreachable.

In this project, `src/index.css` already had support for this idea in the older layout:

```css
#root {
  position: relative;
  z-index: 100001;
  max-width: 100%;
  margin: 0 auto;
  min-height: 100svh;
  display: flex;
  flex-direction: column;
  box-sizing: border-box;
  background: #fff;
}

#root > * {
  flex: 1;
  min-height: 0;
  display: flex;
  flex-direction: column;
}

#root > .luna-root {
  flex: 1 0 auto;
  min-height: min-content;
  height: auto;
}
```

If the similar codebase does not allow page scroll, add equivalent root-level CSS so the footer below the main band is reachable.


STEP 9: FOOTER IMAGE REQUIREMENT
--------------------------------
The app passes:

```ts
footerBackgroundUrl="/news_bg.jpg"
```

So the app must contain:

```text
public/news_bg.jpg
```

If the other codebase uses a different asset, either:
- add that file to `public/`, or
- pass a different URL into `footerBackgroundUrl`.


STEP 10: SUBMODULE WORKFLOW
---------------------------
This setup assumes:
- Sidebar lives in its own repo.
- App points to it as a git submodule.

Simple workflow:

1. Edit sidebar in sidebar repo.
2. Commit and push sidebar repo.
3. In app repo run:

```powershell
.\update-sidebar.ps1
```

or manually:

```bash
git submodule update --remote luna-sidebar
git add luna-sidebar
git commit -m "Bump luna-sidebar submodule"
git push
```

-------------
The key idea is:

- Put footer behavior inside the sidebar repo, not only in the app repo.
- Add `footerBackgroundUrl` to `LunaSidebar`.
